## Entregas

### 1. Guardrail anti-alucinação (commit `54b4ddb`)

Detecção estrutural de narrativa clínica específica no transcript da
Sofia. Quando ela narra dado clínico (idade, PA, glicemia, medicação,
condição, dose, alergia) **sem** ter executado uma tool válida no
turno, **não persiste** no `active_context` cross-channel.

- Flag `_current_response_had_valid_tool` no `GrokCallSession`.
- Reset em `response.created`, mark em `_execute_tool` quando `ok=True`.
- Função `_is_clinical_narration` com 7 padrões regex.
- Log warning `alucination_suspected_skip_persist` pra investigação.

**Por que importa**: a alucinação anterior da Fabiane Beck (30 anos
virou 82 com medicações falsas) contaminou o contexto de futuras
ligações. Agora isso é estruturalmente bloqueado.

### 2. Gate "Ligar Sofia em Alertas" relaxado (commit `339a5e0`)

`alerts-panel.tsx`: botão "Ligar para família (Sofia Voz)" antes
disponível só pra `urgent | critical`. Agora liberado para qualquer
classificação **≠ routine** (inclui `attention`).

Justificativa: queda leve / intercorrência menor também se beneficia
de contato proativo. Sofia mantém o tom adequado via scenario.

### 3. Tools `dial_phone` + `get_patient_responsible_phone` (commit `719a3fc`)

Sofia ganha capacidade de **discar pra terceiros** durante uma
chamada (ex: alertar família).

- `get_patient_responsible_phone`: lê `aia_health_patients.responsible`,
  normaliza pra E.164 BR, retorna lista ordenada de contatos
  (`nurse_override` + `family[]`).
- `dial_phone`: resolve `scenario_code` em `aia_health_call_scenarios`,
  monta payload, faz POST loopback `127.0.0.1:5040/api/voice-call/dial`.
  Bloqueia loop (não pode discar pra si mesma). Passa por Safety
  Guardrail (`action_type=invoke_attendant`).
- Nova config `VOICE_CALL_INTERNAL_URL` (default localhost).

### 4. Métricas Prometheus (commit `a74397a`)

Novo módulo `services/metrics.py` + endpoint `GET /metrics`:

- `voice_call_active_calls{direction}` (gauge)
- `voice_call_calls_started_total{direction}`
- `voice_call_calls_ended_total{direction,state}`
- `voice_call_grok_ws_connected` (gauge)
- `voice_call_tool_executions_total{name,ok}`
- `voice_call_tool_latency_seconds{name}` (histogram)
- `voice_call_grok_first_response_seconds` (slot reservado)
- `voice_call_interrupts_total`
- `voice_call_db_pool_in_use / _max` (coletados on-scrape)
- `voice_call_dial_phone_total{scenario_code,ok}`

Fail-soft: se `prometheus_client` não instalado, hooks viram no-op.

Pontos instrumentados: `routes/dial.py`, `inbound_bridge.py`,
`grok_call_session.py` (WS open/close, interrupt, tool latency).

### 5. Sprint C — Dashboard cross-tenant `/admin/system` (commit `2d3b410`)

Backend: `GET /api/system/dashboard` (super_admin) — 1 query agregando:
- totais globais (tenants ativos/suspensos, pacientes, usuários,
  cuidadores, eventos abertos + 24h)
- série diária de eventos 7d por classification
- top 5 tenants por eventos abertos
- distribuição classification global 30d

Frontend: nova página `/admin/system` com 4 stat cards, mini stacked-bar
7d, lista clickable de top tenants, distribuição percentual 30d.

Sidebar: novo item "Sistema · Cross-tenant" (super_admin only).

### 6. Sprint D — Onboarding tenant wizard `/admin/system/tenants/new` (commit `29e730c`)

Wizard de 5 passos pra provisionar tenant completo:
1. Identidade (slug + nome)
2. IA / Voz (nome, voz Grok, kickoff opcional)
3. Canais (DID, WhatsApp, evolution_instance)
4. Branding (logo, cores)
5. Revisão consolidada

Validação por step (regex slug, campos obrigatórios). Submit reutiliza
`POST /api/system/tenants` existente. Página de listagem ganhou link
"Onboarding wizard" ao lado do form rápido inline.
